Goal:
	Build a directly adressed carthesian mesh that is refined around walls with any number of refinement levels
	This means to setup the following data structure:
	For each grid level:
		
struct DIADGridStruct // DIAD stands for directly adressed
	int cellCountX, cellCountY, cellCountZ, cellCount
		while cellCount represents the actual number of cells that exist and have data, cellCountX, Y, Z informs about theoretical maximum span of the domain.
	int cellCountFineToCoarse, cellCountCoarseToFine
		captures length of the interface
	int gridID
		0 = coarsest
	float res
	float ox, oy, oz
		The origin is always placed in center of a virtual cell [0, 0, 0] that may not actually exist, 
		it is the cell that would be placed in the very corner of the domain, bounded by outer walls of the coarsest grid.
		Thanks to that, the origin can be calculated recursively from the coarser level:
		oxFine = oxCoarse - resFine / 2
	bool bouncebackMarkerArray [zero or cellCount]
		Marks bounceback cells - note that this will be empty (length set to zero) on all grids except for the finest one
	int iArray, jArray, kArray [cellCount]
		This gives position in space as x = ox + i*res, cell [0, 0, 0] has the center placed on [ox, oy, oz]
	DIADEsotwistConnectionsStruct EsotwistConnections
		This gives cell index of the neighbour cell that is located +1 from our cell on the respective axis. This is necessary for esotwist streaming.
	int fineToCoarseWriteArray [cellCountFineToCoarse]
		List of our coarse cells that should receive information from finer grid
	int fineToCoarseReadArray [cellCountFineToCoarse]
		List of fine cells from the finer grid that lie in the left bottom corner of our coarse cell from fineToCoarseWriteArray
	int coarseToFineReadArray [cellCountCoarseToFine]
		List of our coarse cells that should send information to finer grid
	int coarseToFineWriteArray [cellCountCoarseToFine]
		List of fine cells from the finer grid that lie in the left bottom corner of our coarse cell from coarseToFineReadArray

struct DIADEsotwistConnectionsStruct // DIAD stands for directly adressed
	int iNbrArray, jNbrArray, kNbrArray, ijNbrArray, ikNbrArray, jkNbrArray, ijkNbrArray [cellCount]
	
struct DIADNeighboursStruct // DIAD stands for directly adressed
	int iPlusArray, jPlusArray, kPlusArray, iMinusArray, jMinusArray, kMinusArray [cellCount]

How to build the grids?
We are starting with the following information:
	- One or more STLs, inside = solid. Keep it simple that way.
	- getMarkers(ijk) function that might mark bounceback cells in addition to the STLs
	- ox, oy, oz for the coarsest grid
	- resGlobal = res for the coarsest grid
	- using STL dimensions or knowing the domain bounds, we can determine cellCountX, cellCountY, cellCountZ
	- gridLevelCount
	- wallRefinementSpan = how many coarse cells (how thick layer of them) will get refined near wall (2-cell thick interface excluded)?
	This is the input information. From this, we need to start assembling the grid levels. Let's start from the coarsest grid level.

void findMatchingIJKIndex( iWanted, jWanted, kWanted, iSource, jSource, kSource, cellSourceResult )
	Receives sorted iSource, jSource, kSource that !must! be already sorted with key k, j, i
	Loop through Wanted
	for each Wanted, find a matching cell in Source. If its found, write its index to cellSourceResult. If there is no such cell, write -1.

void markWhereFinestFluidIs( fluidMarker, STLs, Info )
	First, do a repetetive fluid marking in order to find which of our coarse cells could possibly contain at least one finest (maximum refinement level) fluid cell. 
	Meaning of this will become clear later. To do this we will be repeatedly temporarily shifting the origin of our grid, to simulate being on a finer grid.
	int shiftCount = 2^(gridLevelCount - gridID - 1) ... 1->1, 2->2, 3->4 etc ... this tells how many times we run the fluid identification
	float finestRes = res / ( 2^(gridLevelCount - gridID - 1) )
	float shiftStart = - res/2 + (finestRes/2)
	for ( shiftCountX = 0; shiftCountX < shiftCount; shiftCountX++ )
		for ( shiftCountY = 0; shiftCountY < shiftCount; shiftCountY++ )
			for ( shiftCountZ = 0; shiftCountZ < shiftCount; shiftCountZ++ )
				create new bounceback array
				add STLs to it
				add marker function to it
				Invert the bounceback array -> now it shows where fluid is
				add the new fluid array to the fluid array of our grid
	
void getDIADEsotwistConnections( Grid )	
	Assemble the domain connections and fill the nbr lists. This includes periodic connection on the borders to allow esotwist to work on borders.
	For all neighbour directions (100, 010, 001):
		Loop through all cells
			keep travelling forward in the given direction and wrap around cellCount[direction], until you find a cell
				assign that cell as the neighbour
	initialize jkNbr, ikNbr, ijNbr, ijkNbr
	Loop through all cells
		Find all neighbours from connectivity of iNbr, jNbr, kNbr, and fill the jk, ik, ij, ijk Nbrs. Connectivity done.
		
void getDIADNeighbours( Grid, Neighbours )	
	Assemble the domain connections and fill the nbr lists. This includes periodic connection on the borders to allow esotwist to work on borders.
	For all neighbour directions (100, 010, 001, -100, 0-10, 00-1):
		Loop through all cells
			travel one step forward in the given direction, if you find a cell, label it as neighbour

Recursive function assembleGrid( grids, level, STLs )
	DIADGridStruct Grid = grids[level]
	
	The input Grid must have already predefined (even though some of it will get overwritten):
		cellCountX, cellCountY, cellCountZ, cellCount, gridID, res, ox, oy, oz
		Most importantly: iArray, jArray, kArray
	
	Sort iArray, jArray, kArray:
		First, sort by k (slowest changing Z coord, second key)
		Next, sort by j, then by i. That are 3 keys in total.
	
	DIADNeighboursStruct Neighbours
	getDIADNeighbours( Grid, Neighbours )
	
	Initialize temporary fluid marker array
	markWhereFinestFluidIs( fluidMarker, STLs, Info )
	
	Now, do this:
		We are now sure that there are not even the finest fluid cells outside of the fluid marked region. 
		However, before throwing off all the other cells, we now we need to add one more layer, in order to be able to place walls next to the fluid again. 
	Initialize temporary borderMarker array
	Loop through all cells. 
		If it is not fluid, check if at least one of its neighbours is fluid. If yes, mark it as border cell.
	
	initialize temporary keepCell array
	If level == gridLevelCount:
	{
		initialize BouncebackMarkerArray
		Loop through all cells.
		If its fluid:
			keep cell.
		Else if its border cell:
			keep cell
			mark it as Bounceback
		Else:
			dont keep it
			
		Now, we will sort cells. We will do it like this:
		First, put cells that will be kept (first key)
		Next, sort by k (slowest changing Z coord, second key)
		Next, sort by j, then by i. That are 4 keys in total.
		All elements in all previously defined arrays must be swapped together with this sort.
	
		Next, count cells that are marked as keep -> reduction -> cellCount (finally!)
		Shorten all existing arrays to length "cellCount"
		
		getDIADEsotwistConnections( Grid )
		return
	}
	
	Now we have marked fluid cells and boundary cells. Next, we need to identify the refinement region.
	initialize temporary fineMarker array
	for layer in range(wallRefinementSpan):
		initialize temporary newFineMarker array
		Loop through all cells.
			If it is border, mark it as newFineMarker
			If its fluid and its neighbour is border, mark it as newFineMarker
			If its fluid and its neighbour is fineMarker, mark it as newFineMarker -> this is how we spread further from the wall each iteration
		fineMarker = newFineMarker
	Now, the layer where we communicate from fine to coarse
	initialize temporary fineToCoarseMarker array
	Loop through all cells.
		If its fluid and its not fineMarker, but its neighbour is fineMarker, mark it as fineToCoarseMarker
	Loop through all cells.
		If its fluid and its not fineMarker and not fineToCoarseMarker, but its neighbour is fineToCoarseMarker, mark it as coarseToFineMarker
	Loop through all cells.
		If its fineToCoarseMarker or coarseToFineMarker, also now mark it as fineMarker.

	Now, assemble the starting point for the fine grid. 
	Reduction: Count fineMarker -> temporary cellCountFine (before the fine cells get filtered by the fine grid)
	It should be 3 plain lists: iFine, jFine, kFine. 
	Copy fineMarker to CPU.
	Initialize iFine, jFine, kFine.
	Serial loop through all cells.
		If its fineMarker, calculate 8 ijk indexes for the 8 fine cells under our coarse cell and write it to iFine, jFine, kFine.
	Copy iFine, jFine, kFine to GPU. We will use it later.

	Now, after assembling the iFine, jFine, kFine instructions, we can delete the cells that we dont need on the coarse grid.
	Right now, we exactly know which cells we need to keep and which not.
	Loop through all cells.
		If its fluid:
			If its not fineMarker, keep cell.
			Else if its fineToCoarse or coarseToFine, keep cell.
			Else, dont keep it
		Else dont keep it
	
	Now, we will sort cells. We will do it like this:
		First, put cells that will be kept (first key)
		Next, sort by k (slowest changing Z coord, second key)
		Next, sort by j, then by i. That are 4 keys in total.
		All elements in all previously defined arrays must be swapped together with this sort.
	
	Next, count cells that are marked as keep -> reduction -> cellCount (finally!)
	Shorten all existing arrays to length "cellCount"
	
	getDIADEsotwistConnections( Grid )
	
	Call the recursive function assembleGrid( iFine, jFine, kFine, Info (?) ) to build the finer grid
	
	After that has been built, the very last thing is that we need indices of its cells where we will communicate.
	The simplest, yet admittedly slow way is to:
	Initialize coarseToFineWriteArray -> same length as coarseToFineReadArray
	For cells in coarseToFineReadArray:
		calculate underlying ijk of the fine grid
		ask the fine grid whats the index of its cell with coords ijk fine
		write the index into coarseToFineWriteArray
	Initialize fineToCoarseReadArray
	For cells in fineToCoarseWriteArray:
		calculate underlying ijk of the fine grid
		ask the fine grid whats the index of its cell with coords ijk fine
		write the index into fineToCoarseReadArray
	END
	
How to update our grid level?
	1) Streaming and then collision on our level
		Read iNbr, jNbr, kNbr, based on those you need to also load ijNbr, ikNbr, jkNbr and ijkNbr
		Perform esotwist and collision on all cells except for stopper cells. -> do I actually need the stopper cells? Could I treat this in a periodic way? -> need to think more
		Bounceback marker array works normally.
		Thanks to our way of IJK indexing, applyMarkers(ijk) will work normally, so boundary conditions get applied the exact same way as before.
	2) Call the finer level to update two times
	3) Perform communication between our grid and the finer grid -> could this be merged into the collision, so that reads are reduced?

	
How to perform communication between a coarse level and a fine level?
	1) Send information from fine to coarse level
		Loop over fineToCoarseWriteArray, find the first read index in fineToCoarseReadArray, find other 7 indexes from connectivity
	2) Send information from coarse to fine level
		Loop over coarseToFineReadArray, find the first write index in coarseToFineWriteArray, find other 7 indexes from connectivity
